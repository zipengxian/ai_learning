# 多语种在线教育平台 - 产品需求文档

## 概述
- **摘要**：构建一个支持英语、日语、韩语等多语种学习的在线教育平台，打造沉浸式语言学习体验。平台涵盖分级课程体系、互动式学习模块（单词记忆、语法练习、口语跟读、听力训练）、学习进度追踪、用户注册登录、个性化学习路径推荐以及社区交流与成就激励系统。
- **目的**：为学习者提供一站式、结构化、互动性强的多语种在线学习环境，降低语言学习门槛，提升学习效率和持续性。
- **目标用户**：希望学习英语、日语、韩语等外语的自主学习用户（学生、职场人士、语言爱好者）。

## 目标
- 构建支持课程分级浏览与学习的课程体系模块
- 实现四大互动学习模块（单词记忆、语法练习、口语跟读、听力训练）
- 实现用户注册、登录与身份认证体系
- 实现个人学习进度追踪与数据看板
- 实现基于用户水平的个性化学习路径推荐
- 构建社区交流（讨论区）与成就徽章激励系统

## 非目标（超出范围）
- 实时一对一在线授课/视频直播功能
- 移动端原生 App（仅限 Web 响应式）
- 支付与付费课程体系
- 第三方社交账号登录（如微信、Google OAuth）
- AI 语音评测引擎自研（可使用浏览器 Speech API 或模拟方案）
- 课程内容管理系统（CMS）后台——课程数据通过数据库种子脚本预置

## 背景与上下文
- 平台采用前后端分离架构：React 前端 + Node.js/Express 后端 + SQLite 数据库（轻量级，适合快速原型）
- 口语跟读功能基于浏览器 Web Speech API 实现语音识别
- 课程数据采用种子数据方式预置，覆盖英语、日语、韩语三个语种，每个语种至少三个难度级别
- 个性化推荐基于用户当前学习等级和历史完成数据进行简单规则匹配

## 功能需求
- **FR-1**：用户可通过邮箱和密码完成注册，并通过邮箱和密码登录系统
- **FR-2**：系统提供英语、日语、韩语课程的分级浏览（入门、初级、中级），每级别包含多个课程单元
- **FR-3**：每个课程单元包含四类互动学习活动：单词记忆（闪卡模式）、语法练习（选择题）、口语跟读（语音识别）、听力训练（音频播放+理解题）
- **FR-4**：系统记录用户在每个课程单元中各类活动的完成状态和得分
- **FR-5**：用户可在个人看板查看整体学习进度（完成课程数、学习天数、各技能得分分布）
- **FR-6**：系统根据用户当前等级与未完成课程，生成推荐学习路径
- **FR-7**：用户可在社区讨论区发帖、回帖，进行学习交流
- **FR-8**：系统根据用户学习行为（连续学习天数、完成课程数、积分等）自动授予成就徽章

## 非功能需求
- **NFR-1**：页面首次加载时间（FCP）< 3秒
- **NFR-2**：支持主流浏览器（Chrome, Firefox, Edge 最新两个版本）
- **NFR-3**：用户密码采用 bcrypt 哈希存储，不存储明文密码
- **NFR-4**：API 响应时间 P95 < 500ms（除语音识别外部依赖外）
- **NFR-5**：界面支持响应式布局，适配桌面端和移动端

## 约束
- **技术栈**：React 18+ (前端)、Node.js + Express (后端)、SQLite (数据库)
- **业务约束**：课程数据通过种子脚本预置，不开发 CMS 后台
- **依赖**：口语跟读依赖浏览器 Web Speech API（仅 Chrome 和 Edge 完整支持）

## 假设
- 用户具备基本的计算机操作能力，能够使用浏览器访问平台
- 课程内容（单词、语法题、听力音频等）由产品方提供，开发阶段使用示例数据
- 语音识别准确率依赖浏览器内置引擎，平台不自行训练模型
- 用户学习行为数据足够支撑简单的规则推荐，无需复杂机器学习模型

## 验收标准

### AC-1：用户注册与登录
- **Given**：用户访问平台注册页面
- **When**：用户输入有效邮箱和密码（≥6位）并提交注册
- **Then**：系统创建账户并自动跳转至登录后首页，后续可使用该邮箱和密码登录
- **Verification**：`programmatic`
- **Notes**：需验证重复邮箱注册拒绝、密码长度校验、登录失败提示

### AC-2：课程分级浏览
- **Given**：用户已登录
- **When**：用户进入课程中心，选择语种（如英语）和级别（如初级）
- **Then**：展示该级别下所有课程单元列表，每个单元显示标题和简短描述
- **Verification**：`programmatic`

### AC-3：单词记忆（闪卡模式）
- **Given**：用户进入某个课程单元的单词记忆活动
- **When**：用户浏览单词闪卡（正面显示单词，点击翻转显示释义），逐张浏览并标记"已掌握"或"需复习"
- **Then**：系统记录用户对该单词的掌握状态，完成后显示本轮正确率
- **Verification**：`programmatic`

### AC-4：语法练习（选择题）
- **Given**：用户进入语法练习活动
- **When**：用户逐题作答（单选题），提交答案后显示正确/错误反馈
- **Then**：完成全部题目后显示总分与正确率，结果写入学习记录
- **Verification**：`programmatic`

### AC-5：口语跟读
- **Given**：用户进入口语跟读活动，浏览器支持 Speech API
- **When**：用户点击麦克风按钮朗读屏幕展示的句子，系统调用语音识别并对比原文
- **Then**：显示识别结果与原文的匹配度，并记录完成状态
- **Verification**：`programmatic`
- **Notes**：不支持的浏览器需提示"您的浏览器不支持语音识别"

### AC-6：听力训练
- **Given**：用户进入听力训练活动
- **When**：用户点击播放音频，听完后回答理解题（选择题）
- **Then**：提交后显示得分，完成状态写入学习记录
- **Verification**：`programmatic`

### AC-7：学习进度看板
- **Given**：用户已登录且有学习记录
- **When**：用户进入个人看板页面
- **Then**：展示总完成课程数、连续学习天数、各技能（词汇/语法/口语/听力）得分分布图表
- **Verification**：`programmatic`

### AC-8：个性化学习路径推荐
- **Given**：用户已登录，系统有用户等级和学习记录
- **When**：用户进入首页或推荐页面
- **Then**：展示推荐的下一个课程单元，基于用户当前等级和未完成课程
- **Verification**：`programmatic`

### AC-9：社区讨论区
- **Given**：用户已登录
- **When**：用户进入社区页面，可浏览帖子列表、发新帖、查看帖子详情并回帖
- **Then**：帖子按发布时间倒序展示，支持分页浏览
- **Verification**：`programmatic`

### AC-10：成就徽章激励
- **Given**：用户完成特定行为（如连续学习7天、完成10门课程、获得100积分）
- **When**：满足成就条件后
- **Then**：系统自动授予对应徽章，在用户个人主页展示已获得徽章
- **Verification**：`programmatic`
