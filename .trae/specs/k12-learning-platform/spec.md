# 中小学学习平台 - 产品需求文档

## 概述
- **摘要**：构建一个覆盖中小学全学科（语文、数学、英语、科学、历史、地理等）的综合学习平台，支持 Web 端、移动端（手机/平板）和桌面端三端使用。平台参考 Trae 工作平台的设计语言，采用左侧导航/资源面板 + 中央内容区 + 右侧 AI 助手的侧边栏布局，打造沉浸式、AI 辅助的现代化学习体验。
- **目的**：为学生提供一站式全学科学习入口，涵盖教材内容、练习题库、视频课程、AI 答疑辅导等功能，消除多 App 切换的碎片化体验。
- **目标用户**：中小学生（6-18 岁）及其家长/教师。

## 目标
- 构建全学科课程体系，覆盖 K-12 阶段主要学科
- 实现三端（Web / 移动端 / 桌面端）一致的学习体验
- 参考 Trae 工作平台设计语言，打造侧边栏 + 内容区 + AI 面板的三栏式布局
- 嵌入 AI 学习助手，支持知识问答、题目讲解、作文批改等场景
- 支持学习进度追踪、错题本、学习报告
- 支持离线学习（移动端/桌面端缓存课程内容）

## 非目标（超出范围）
- 不开发直播授课系统（仅点播/录播）
- 不开发家校互通/家长端（仅学生端，后续可扩展）
- 不开发教师后台排课系统（课程结构通过配置文件预置）
- 不做 AI 模型自训练，接入第三方 LLM API（如 DeepSeek/通义千问）
- 游戏化社交功能（如好友排行榜）—— 此版本暂不包含

## 背景与上下文
- 平台参考 Trae 工作平台 UI 设计模式：
  - **左侧面板**：学科导航 / 资源浏览器 / 学习大纲
  - **中央内容区**：教材阅读、视频播放、练习作答
  - **右侧面板**：AI 学习助手聊天窗口（可收起）
  - **顶部栏**：用户信息、搜索、设置、主题切换
- 三端策略：Web 端 Vite + React，移动端 React Native (Expo)，桌面端 Electron（复用 Web 端代码）
- AI 助手采用流式响应（SSE），支持 Markdown 和 LaTeX 渲染
- 课程内容通过结构化 JSON/数据库预置，支持按年级 - 学科 - 单元 - 知识点四级组织

## 功能需求

### 核心学习
- **FR-1**：按年级（1-9 年级 / 高中 1-3 年级）和学科（语文/数学/英语/物理/化学/生物/历史/地理）筛选课程内容
- **FR-2**：支持教材章节式的学习大纲浏览，每章节包含知识点讲解、例题、练习题
- **FR-3**：练习题支持选择题、填空题、判断题、简答题等多种题型，自动批改并记录
- **FR-4**：视频课程播放（支持倍速、进度记忆）
- **FR-5**：自动生成错题本，按学科和知识点归类

### AI 学习助手
- **FR-6**：AI 助手以右侧面板形式常驻，用户可随时提问
- **FR-7**：AI 支持学科知识问答、解题思路讲解、作文批改建议
- **FR-8**：AI 回答支持 Markdown 渲染（代码块、公式 LaTeX、图表）
- **FR-9**：AI 可感知当前学习上下文（当前章节/题目），提供针对性辅导
- **FR-10**：支持对话历史保存和回看

### 用户系统
- **FR-11**：用户注册登录（邮箱/手机号 + 密码），支持多设备登录
- **FR-12**：用户设置年级和学科偏好，首页展示对应课程
- **FR-13**：学习记录云端同步，跨设备进度一致

### 三端适配
- **FR-14**：Web 端完整三栏布局，桌面端 Electron 与 Web 端一致体验
- **FR-15**：移动端适配触控操作，侧边栏改为底部 Tab 导航，AI 助手改为全屏会话
- **FR-16**：移动端/桌面端支持离线缓存已学习内容

### UI/UX（参考 Trae 设计）
- **FR-17**：整体采用"侧边栏导航 + 中央内容区 + 右侧 AI 面板"三栏布局
- **FR-18**：支持明暗主题切换，默认跟随系统
- **FR-19**：内容区采用卡片式设计，合理的留白和间距
- **FR-20**：AI 面板可拖拽调整宽度，可收起/展开
- **FR-21**：顶部搜索栏支持全局搜索（课程/知识点/题目）

## 非功能需求
- **NFR-1**：页面首次加载时间 < 2s（Web/桌面端）
- **NFR-2**：AI 助手首 Token 响应时间 < 1.5s（流式）
- **NFR-3**：移动端安装包 < 50MB，桌面端安装包 < 100MB
- **NFR-4**：支持离线状态下浏览已缓存内容
- **NFR-5**：用户数据加密存储，符合未成年人数据保护要求

## 约束
- **技术栈**：
  - Web/桌面端：React 18+ + TypeScript + Vite
  - 移动端：React Native (Expo)
  - 桌面端壳：Electron
  - 后端：Node.js + Express + PostgreSQL
  - AI 接入：SSE 流式 + LLM API（DeepSeek / 通义千问）
  - 状态管理：Zustand
  - UI 组件：自建组件库（风格统一）
- **团队**：中小型团队，以 React 技术栈为主

## 假设
- 学生具备基本的电脑/平板操作能力
- 课程内容由合作教研团队提供，开发阶段使用示例数据
- AI 能力依赖第三方 LLM API，在 API 不可用时降级为预设问答库
- 用户家庭具备基本的网络条件

## 验收标准

### AC-1：三栏布局与导航
- **Given**：用户登录平台（Web/桌面端）
- **When**：首页加载完成
- **Then**：左侧显示学科导航面板，中央显示内容区，右侧显示 AI 助手面板（可收起）
- **Verification**：`human-judgment`

### AC-2：学科内容浏览
- **Given**：用户已登录
- **When**：用户选择年级和学科
- **Then**：展示该学科的大纲目录，点击章节展开知识点列表
- **Verification**：`programmatic`

### AC-3：练习题作答与批改
- **Given**：用户进入练习题页面
- **When**：用户完成题目并提交
- **Then**：系统自动批改，显示正确/错误，并记录到错题本
- **Verification**：`programmatic`

### AC-4：AI 助手问答
- **Given**：AI 助手面板已打开
- **When**：用户输入问题并发送
- **Then**：AI 流式返回回答，支持 Markdown/LaTeX 渲染
- **Verification**：`programmatic`

### AC-5：AI 上下文感知
- **Given**：用户正在学习某章节内容
- **When**：用户点击"问 AI"或发送问题
- **Then**：AI 的回复自动关联当前章节/题目上下文
- **Verification**：`human-judgment`

### AC-6：错题本
- **Given**：用户有练习记录
- **When**：用户进入错题本页面
- **Then**：展示所有错题，按学科和知识点归类，支持重新练习
- **Verification**：`programmatic`

### AC-7：明暗主题切换
- **Given**：用户在任何页面
- **When**：用户点击主题切换按钮
- **Then**：页面在明暗主题间切换，所有组件适配正确
- **Verification**：`human-judgment`

### AC-8：移动端适配
- **Given**：用户在手机端访问
- **When**：应用加载
- **Then**：底部显示 Tab 导航（首页/课程/AI/我的），侧边栏转为抽屉式
- **Verification**：`human-judgment`

### AC-9：桌面端 Electron 打包
- **Given**：桌面端 Electron 应用已打包
- **When**：用户安装并启动
- **Then**：应用窗口显示，功能与 Web 端一致
- **Verification**：`programmatic`

### AC-10：跨设备进度同步
- **Given**：用户在不同设备登录同一账号
- **When**：在 A 设备完成学习后，打开 B 设备
- **Then**：B 设备显示最新的学习进度和错题数据
- **Verification**：`programmatic`

### AC-11：离线缓存
- **Given**：用户已缓存某章节内容
- **When**：设备断网后进入该章节
- **Then**：可正常浏览已缓存的内容，AI 助手提示"网络已断开"
- **Verification**：`programmatic`
